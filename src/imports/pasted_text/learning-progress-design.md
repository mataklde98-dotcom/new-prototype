Improve the visual design of the Learning Progress screen to make it cleaner, calmer, and easier to scan for students.

The current structure (Subject → Category → Topic) should remain the same, but the interface should be visually simplified.

The goal is to reduce visual clutter and make the most important information immediately understandable.

⸻

1. Simplify the Subject Cards

Each subject card should focus on only a few key elements.

Display only:

• Subject name
• Overall progress percentage
• Progress circle indicator
• Trend indicator (up or down)

Optional secondary information:

• strongest category (small text)

Remove or minimize additional labels such as attention badges unless absolutely necessary.

Subject cards should feel light and easy to scan.

⸻

2. Reduce Visual Noise

Avoid showing too many visual indicators at the same time.

Limit the UI to:

• one circular progress indicator for subjects
• horizontal progress bars for categories
• percentage labels

Avoid combining multiple indicators such as:

• circles
• bars
• badges
• counters
• multiple colors

in the same visual block.

⸻

3. Use a Simpler Color System

Use a minimal color system:

• green → strong progress
• yellow → moderate progress
• red → needs improvement
• neutral gray → background

Avoid using many different accent colors.

The interface should feel calm and consistent.

⸻

4. Improve Spacing and Readability

Increase vertical spacing between:

• subject cards
• category rows
• progress bars

The layout should breathe more and avoid a crowded feeling.

⸻

5. Keep Categories Collapsible

Categories should remain collapsible.

Default state:

Show only category rows with progress bars.

Example:

Numbers & Operations — 79%
Functions — 58%
Geometry — 72%
Statistics — 48%

Topics should appear only when the category is expanded.

This keeps the screen compact and easy to scan.

⸻

6. Focus on Scanability

The student should be able to understand the screen in under 3 seconds:

• which subjects they are doing well in
• which areas need improvement
• where they should continue learning

The design should prioritize clarity and calmness over decorative UI elements.
